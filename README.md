# 🎧 Telegram Bot — Система кодов для сведения

Простой Telegram-бот с системой кодов, проверкой подписки и админ-панелью.

## 📁 Файлы

```
bot.py            — весь бот (один файл)
requirements.txt  — зависимости
.env              — настройки (создай сам из .env.example)
db.json           — база данных (создаётся автоматически)
```

## ⚙️ Настройка

### 1. Установи зависимости

```bash
pip install -r requirements.txt
```

### 2. Создай .env файл

Скопируй `.env.example` → `.env` и заполни:

```env
BOT_TOKEN=        # токен от @BotFather
ADMIN_ID=         # твой Telegram ID (узнай через @userinfobot)
CHANNEL_ID=       # @username канала ИЛИ числовой ID (например -1001234567890)
CHANNEL_LINK=     # https://t.me/...
WORKS_CHANNEL_LINK= # https://t.me/...
DM_LINK=          # https://t.me/твой_username
```

### 3. Сделай бота администратором канала

Чтобы проверка подписки работала — добавь бота в канал как администратора.

### 4. Запусти

```bash
python bot.py
```

| Команда  | Описание             |
|----------|----------------------|
| `/start` | Главное меню         |
| `/admin` | Админ-панель (только для ADMIN_ID) |

---

## 🔑 Как работают коды

1. Ты создаёшь код в `/admin` → **Создать код**
2. Бот генерирует номер (00001–99999)
3. Ты выбираешь тип контента: текст / фото / видео / файл / ссылка
4. Отправляешь контент
5. Код сохранён! Пользователь вводит его → получает контент

## 🚂 Деплой на Railway

1. Загрузи проект на GitHub
2. Создай новый проект на [railway.app](https://railway.app)
3. Подключи репозиторий
4. Добавь переменные из `.env` в Settings → Variables
5. Railway запустит бота автоматически

> **Важно:** на Railway `db.json` сбрасывается при каждом деплое.  
> Для постоянного хранения подключи Railway Volume или используй внешнюю БД.
